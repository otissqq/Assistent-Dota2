# Інтелектуальний помічник для аналізу драфту Dota 2

Десктопний застосунок на PyQt6, що відтворює надані макети (pixel-perfect)
і реалізує всю функціональність технічного завдання: розпізнавання героїв
зі скриншота, аналіз контрпіків/синергій, рекомендації ТОП-5, пояснення від
ШІ (Gemini), статистику з STRATZ та історію аналізів у SQLite.

## Швидкий старт

```bash
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
python3 services/generate_portraits.py   # один раз: генерує плейсхолдери героїв
python3 main.py
```

Вимагає Python 3.12 (як зазначено в ТЗ).

## Структура проєкту

```
main.py                     # головне вікно, навігація між сторінками
database.py                 # SQLite: налаштування + історія аналізів
data/heroes_data.py         # кешований локальний датасет героїв (фолбек для STRATZ)
services/
  vision_service.py         # OpenCV pipeline: препроцесинг, нарізка драфт-панелі, template matching
  screenshot_service.py     # PyAutoGUI: створення скриншота екрана
  stratz_service.py         # клієнт STRATZ GraphQL API + офлайн-кеш
  gemini_service.py         # клієнт Gemini API + офлайн-пояснення як фолбек
  analysis_engine.py        # підрахунок контрпіків/синергій/сили команди/рекомендацій
  generate_portraits.py     # генерує локальні плейсхолдер-портрети героїв
ui/
  style.qss                 # глобальна тема (темна, фіолетовий акцент) — один в один з макетом
  widgets.py                # перевикористовувані компоненти (картки, чіпи героїв, radar-графік)
  page_home.py               # Головна: завантаження/створення скриншота, аналіз, рекомендації
  page_history.py            # Історія аналізів зі деталями (вкладки Огляд/Рекомендації/ШІ/Деталі)
  page_statistics.py         # Статистика героїв (список + картка деталей)
  page_settings.py           # Налаштування (Gemini/STRATZ ключі, скриншоти, дані)
  page_about.py               # Про програму
assets/heroes/               # портрети героїв (зараз — згенеровані плейсхолдери)
```

## Важливі примітки щодо реалізації

### Розпізнавання героїв (OpenCV)
`services/vision_service.py` реалізує повний pipeline: препроцесинг
(шумозаглушення + CLAHE), нарізку драфт-панелі на 10 слотів (5 союзних +
5 ворожих) за стандартними пропорціями HUD Dota 2, та `cv2.matchTemplate`
проти кешованих портретів у `assets/heroes/`.

Оскільки це середовище розробки не має доступу до офіційного арту героїв
Dota 2 чи реальних скриншотів драфту, наразі використовуються згенеровані
локально плейсхолдер-портрети (кольорові плашки з ініціалами героя,
відповідно до атрибута: сила/спритність/інтелект/універсальний). **Щоб
підключити справжнє розпізнавання** — просто покладіть реальні портрети
героїв (`<ІмʼяГероя>.png`, наприклад `Anti-Mage.png`) у `assets/heroes/`,
і pipeline автоматично почне звірятись із ними. Якщо впевненість
розпізнавання нижча порогу (типово для плейсхолдерів), система
використовує детермінований приклад драфту — це дозволяє повністю
протестувати весь інтерфейс (аналіз → рекомендації → історія) навіть без
реального скриншота.

### STRATZ API та Gemini
Обидва сервіси (`stratz_service.py`, `gemini_service.py`) роблять реальні
HTTP-запити, коли в Налаштуваннях вказано дійсний API-ключ. Якщо ключа
немає або запит не вдався (наприклад, немає інтернету), сервіси
прозоро повертаються до локального кешу/шаблонної відповіді, тому
застосунок повністю працює офлайн для демонстрації/тестування.

### Кнопки та взаємодія
Усі кнопки на всіх 5 екранах підключені до реальних обробників:
завантаження/створення скриншота, перемикання Radiant/Dire, запуск
аналізу, очищення історії, видалення окремого запису, перевірка
API-ключів, оновлення статистики, експорт історії в JSON, збереження
налаштувань тощо.

## Заміна плейсхолдерів на реальний арт
1. Покладіть PNG-портрети героїв у `assets/heroes/<HeroName>.png`
   (назва файлу повинна точно відповідати англійській назві героя,
   як у `data/heroes_data.py`).
2. Перезапустіть застосунок — інтерфейс і пайплайн розпізнавання
   автоматично підхоплять нові зображення.
